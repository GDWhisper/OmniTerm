## 1. 环境准备

### 1.1 系统依赖

| 依赖 | 最低版本 | 用途 |
|------|---------|------|
| Rust | 1.87+ | 后端编译 |
| Node.js | 22+ | 前端构建 |
| pnpm | 10+ | 前端包管理 |
| tmux | 3.3+ | 终端会话管理 |
| SQLite | 3.35+ | 数据存储 |

### 1.2 启动开发环境

**一键启动（推荐）：**

```bash
cd /home/pax/coding/OmniTerm

./dev.sh start     # 启动后端 + 前端
./dev.sh stop      # 停止所有服务
./dev.sh restart   # 重启
./dev.sh status    # 查看运行状态
./dev.sh logs      # 实时日志（Ctrl+C 退出）
./dev.sh logs be   # 仅后端日志
./dev.sh logs fe   # 仅前端日志
```

启动成功后输出：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  后端 API:  http://localhost:9777
  前端 UI:   http://localhost:9778
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

> 后端仅监听 127.0.0.1（通过 Vite proxy 访问），前端监听 0.0.0.0（局域网可访问）。

**手动启动：**

```bash
# 终端 1：后端
cd /home/pax/coding/OmniTerm
. "$HOME/.cargo/env"
cargo run
# → Listening on 0.0.0.0:9777

# 终端 2：前端
cd frontend
pnpm dev
# → Vite dev server on http://localhost:9778 (proxies /api → :9777)
```

### 1.3 验证服务

```bash
curl http://localhost:9777/api/v1/health
# 期望：{"status":"ok"}
```

## 2. 首次启动

### 2.1 设置管理员密码

1. 打开浏览器访问 `http://localhost:9778`
2. 系统检测到无管理员账号，显示密码设置界面
3. 输入密码并提交
4. 自动登录进入主界面

**验证点：**
- [ ] 密码设置成功后自动登录
- [ ] 密码为空时提示错误
- [ ] 刷新页面后仍保持登录状态

### 2.2 主界面布局

首次进入应看到三栏布局：

```
┌──────────┬────────────────────────┬──────────────┐
│  侧边栏   │       终端区域          │   文件管理器  │
│ 140-280px│       flex-1           │  240-400px   │
│          │                        │              │
│ OmniTerm │  Select or create      │  Select a    │
│          │  a session             │  workspace   │
│          │                        │              │
└──────────┴────────────────────────┴──────────────┘
```

**验证点：**
- [ ] 三栏同时可见
- [ ] 侧边栏与文件管理器之间有拖拽分割线
- [ ] 拖拽分割线可调整面板宽度（侧边栏 140-280px，文件管理器 240-400px）
- [ ] 面板宽度刷新后保持（localStorage 持久化）

## 3. 工作区管理

### 3.1 创建工作区

1. 点击侧边栏顶部 **+** 按钮
2. 输入工作区名称（如 `Home`）
3. 输入根路径（如 `/home/pax`）
4. 确认

**验证点：**
- [ ] 工作区出现在侧边栏列表中
- [ ] 显示名称和根路径
- [ ] 空名称被拒绝
- [ ] 不存在的路径被拒绝（后端校验）

### 3.2 切换工作区

- 点击工作区名称展开/折叠
- 选中的工作区高亮显示
- 文件管理器自动加载该工作区根目录

**验证点：**
- [ ] 点击切换展开状态
- [ ] 文件管理器内容随工作区切换更新
- [ ] 终端区域不受影响

### 3.3 删除工作区

（当前 UI 未暴露删除按钮，通过 API 测试）

```bash
curl -X DELETE http://localhost:9777/api/v1/workspaces/{id}
```

## 4. 终端会话

### 4.1 创建会话

1. 展开一个工作区
2. 点击 Sessions 旁的 **+** 按钮
3. 可选输入会话名称
4. 确认

**验证点：**
- [ ] 会话出现在工作区下方
- [ ] 会话前有绿色圆点指示器
- [ ] 无名称时显示 tmux 自动生成的名称

### 4.2 连接终端

1. 点击一个会话
2. 终端区域建立 WebSocket 连接
3. 显示 `[attached to session-name]` 绿色提示
4. 出现 shell 提示符

**验证点：**
- [ ] 连接成功显示绿色提示
- [ ] 可以正常输入命令（如 `ls`, `pwd`）
- [ ] 输出实时显示，无明显延迟
- [ ] 终端窗口大小自动适配面板
- [ ] 拖拽调整面板宽度后终端自动重新适配

### 4.3 终端基本操作

在终端中执行以下操作：

```bash
# 基本命令
echo "Hello OmniTerm"
ls -la
cat /etc/hostname

# 交互式命令
top        # 按 q 退出
vim        # 正常编辑，:q 退出
htop       # 如果已安装

# 长输出
find / -name "*.conf" 2>/dev/null | head -50
```

**验证点：**
- [ ] 命令输出正确
- [ ] 交互式程序（vim/htop）正常工作
- [ ] 颜色输出正确渲染
- [ ] 长输出可滚动查看
- [ ] Ctrl+C 中断正常工作

### 4.4 多会话

1. 创建第二个会话
2. 在两个会话之间切换
3. 每个会话保持独立状态

**验证点：**
- [ ] 切换会话后终端内容保持
- [ ] 返回原会话时状态完整
- [ ] 不同会话可以运行不同程序

### 4.5 断开与重连

1. 刷新浏览器页面
2. 重新点击同一会话

**验证点：**
- [ ] 显示 `[disconnected]` 红色提示
- [ ] 重连后显示 `[attached to ...]` 绿色提示
- [ ] tmux 会话内容仍然存在（持久化）

## 10. 已知限制

| 缺陷 | 影响 | 规避方案 |
|------|------|---------|
| 亮色主题下终端背景仍为深色 | 亮色模式下终端区域视觉不一致 | 设计上刻意区分，终端保持深色护眼 |
| 文件管理器不支持内联编辑 | 需要下载后编辑再上传 | 使用终端中的 vim/nano 等编辑器 |
| Agent Hook 状态未在前端展示 | Hook 扫描功能后端就绪但前端无 UI | 后续迭代加入状态指示器 |
| 无文件搜索 UI | 后端 search API 就绪，前端未集成 | 后续迭代加入搜索框 |
| 无文件预览 | cubone 组件预览已禁用 | 后续可启用文本/图片预览 |
| 无 HTTPS | 生产环境需要反向代理 | 使用 Nginx/Caddy 配置 TLS |
