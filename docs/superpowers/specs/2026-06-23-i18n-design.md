# i18n 国际化设计

**日期**: 2026-06-23
**状态**: 待审批
**范围**: 前端 UI 静态文本 + 终端连接状态消息国际化

---

## 目标

为 OmniTerm 前端添加中英双语支持：
- 默认语言跟随浏览器/系统语言
- 用户可在设置中手动切换
- 切换语言后 UI 即时刷新，无需刷新页面

## 技术选型

**react-i18next** — 基于 i18next 的 React i18n 方案

- `useTranslation()` hook + `t('key')` 函数
- `i18next-browser-languagedetector` 自动检测浏览器语言
- 翻译文件按组件前缀组织的扁平 key
- 插值语法：`{{variable}}`

**不引入的方案**：
- Lingui（编译时提取，对小项目过重）
- 自定义方案（需自己实现插值/复数/检测，重复造轮子）

## 语言检测与持久化

优先级从高到低：

1. `localStorage('omniterm_locale')` — 用户手动选择过
2. `navigator.language` / `navigator.languages` — 浏览器/系统语言
3. `'en'` — 最终回退

i18next 的 `LanguageDetector` 插件自动处理检测和读写 localStorage，不需要在 Zustand 中管理语言状态。

### 为什么不加入 Zustand

语言状态由 i18next 自行维护（当前语言、资源加载、`t()` 绑定）。组件通过 `useTranslation()` 自动订阅变化并重渲染。如果再用 Zustand 存一份，会产生双重状态同步问题。

## 翻译文件结构

```
frontend/src/
  i18n.ts                    # i18next 初始化配置
  locales/
    zh/
      translation.json       # 中文翻译
    en/
      translation.json       # 英文翻译
```

### Key 命名规则

- `.` 分隔的扁平 key，前缀对应组件/模块
- 插值用 `{{variable}}` 语法
- 纯功能性单位（`"px"`、`"项"`）不翻译

### 前缀分配

| 前缀 | 对应组件 |
|------|---------|
| `sidebar.*` | Sidebar.tsx |
| `settings.*` | Settings.tsx |
| `terminal.*` | Terminal.tsx |
| `terminal.status.*` | useTerminal.ts |
| `fm.*` | FileManager.tsx |
| `drawer.*` | FileDrawer.tsx |
| `preview.*` | FilePreview.tsx |
| `mobile.*` | MobileNav.tsx |
| `modal.*` | ConfirmDialog.tsx |

## Settings 语言选择器 UI

在"主题"下方新增"语言"选项，交互风格与主题选择一致：

```
┌─────────────────────────────┐
│  设置                       │
│                             │
│  主题                       │
│  [☀️ 亮色] [🌙 暗色] [💻 跟随系统] │
│                             │
│  语言                       │
│  [中文]  [English]          │
│                             │
│  终端字体大小               │
│  [−] ──●────── [+] 14px    │
│                             │
│  关于                       │
│  OmniTerm -- Web-based...   │
└─────────────────────────────┘
```

- 两个按钮，选中态与主题按钮一致
- 点击立即切换，所有 UI 文本即时刷新
- 语言按钮文案直接写死（中文按钮显示"中文"，英文按钮显示"English"），不走翻译

## 不翻译的内容

- 后端 API 返回的错误/成功消息
- `"px"`、`"项"` 等单位符号
- 产品名 `"OmniTerm"`
- 版本号、License 信息
- 终端 ANSI 消息中的变量部分（session name、exit code 等）

## 设计决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| i18n 库 | react-i18next | 生态成熟、内置语言检测和持久化、插值开箱即用 |
| 回退语言 | `en` | 英文是更通用的回退语言 |
| Key 组织 | 扁平 + 组件前缀 | 简单直接，按前缀即可定位组件 |
| 语言状态管理 | i18next 自管，不入 Zustand | 避免双重状态同步 |
| 语言按钮文案 | 写死不翻译 | 避免鸡生蛋问题 |
| 终端状态消息 | 纳入 i18n | 用户选择翻译范围包括终端连接消息 |
